newT.js

Tasty JavaScript Templating.

You love building JavaScript apps. Armed with the latest tools, you're dying for a tasty little JS template that has a friendly syntax, low learning curve and lots of room to grow. Enter newT.js, tasty JavaScript templating.

With a nod to Haml, and much love to super templates like mustache.js, newT.js is a pure JS templating engine with the power and flexibility you need. newT.js helps you build powerful, responsive, asynchronous JavaScript applications. newT.js syntax elimantes tedious closing and balancing of HTML tags and it's tiny [4.6KB], while providing flexible template options. And, newT.js helps you build a safer site, aiding in the fight against persistent XSS attacks.

How do you get all these awesome features? Easy, follow the instructions below for tasty JS templating

Add newT.js

To get started, add newT.js to your page. 

<script src="newT.js" type="text/javascript"></script>

Using newT.js is simple, just 3 basic steps

  1. Save a Template
  2. Create a DOM Elememt
  3. Render a Template

Step #2 is technically a substep of step #1, but that's splitting hairs and we only care about fast templating. Let's continue.

Save a newT.js Template

No overiew or tutorial would be complete without a cheesy hello world example. Let's start with a simple template, a DIV with some dynamically injected text for flavor.

newT.save("helloworld", function(data){
    return newT.div( data.text );
});

Wow, that's a lot of code! Well, not really, but let's review what just happened. the newT.save gets two arguments. The first is the name of the template. We can reference any newT.js template later using this name.

The second argument is an anonymous function, we'll use it later to pass in some dynamic data. This anonymous function will return our hello world template, ready for DOM insertion. Here, we are using a simple DIV, let's not get too crazy, yet!

Render a newT.js Template

Onto step #3, rendering the cheesy hello world example.

document.body.appendChild( 
    newT.render( "helloword" { 
        text : "hello newT world!" 
    })
);

Adds a new DIV element to the body element with the hello world text.

<div>hello newT world!"</div>

Pretty painless, right? Since newT.js always returns a valid DOM fragment or element, it's easy to just append it to any element, in this case, the body tag.

Going Deeper with newT.js

Rendering a single element is great, but it's much more likely you'll need to create a more complex HTML structure. Let's make a more robust hello world.

First, save a new template.

newT.save("my_template", function(t_data){
    return (
        newT.div({clss : "outerdiv" }, "Hello World!", 
            newT.div( {clss : "innerdiv1" }, t_data.foo ), 
            newT.div( t_data.bar )
        )
    )
});
var my_data={ foo : "newT hello world", bar : "w00t!" };
document.body.appendChild( 
    newT.render("my_template", my_data ) 
);

Let's go line by line. Saving a template looks the same, give it a name and a render method. This method needs to return a DOM element or fragment.

This return is a little more complex looking, but it's also very simple. Our first element, has a CSS class of "outerdiv". The second argument is the string we want to appear in side the DIV. The 3rd and 4th arguments will be children elements of "outerdiv".

<div class="outerdiv">Hello World!
    <div class="innerdiv1">newT hello world</div>
    <div>w00t!</div>
</div>
